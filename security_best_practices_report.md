# ÉCLAT Nail Atelier — security review

Проверено 7 июля 2026 года.

## Итог

Технические замечания высокого и среднего риска исправлены. Публичная раздача построена на allowlist, Mini App проверяет подпись Telegram, административный вход и заявки ограничены по частоте, HTTP-тело ограничено по размеру, CSP и защитные заголовки включены. `npm audit` не обнаружил известных уязвимостей.

## Исправлено

- SEC-002: обработчик POST больше не накапливает тело после лимита 100 000 байт и заранее проверяет `Content-Length`.
- SEC-003: production runtime переведён с Node.js 20 на Node.js 24.
- SEC-004: новые booking/review ID используют 8 случайных байт; клиентская ссылка дополнительно содержит 96-битный секрет.
- SEC-005: истёкшие rate-limit buckets очищаются каждые пять минут, размер карты ограничен.
- SEC-006: статические ресурсы принимают только `GET` и `HEAD`; остальные методы получают `405`.

## Подтверждённые защиты

- `.env`, исходный сервер, Git-метаданные, журналы и данные записей недоступны по HTTP.
- `.env`, `data/booking-data.json`, журналы и Playwright-артефакты исключены из Git.
- Mini App API без корректной подписи Telegram возвращает `401`.
- CSP запрещает inline/eval-скрипты, framing и произвольные источники.
- Данные отзывов выводятся через `textContent`; значения Mini App экранируются.
- Большое тело POST получает `400`, после чего `/health` продолжает отвечать.
- `DELETE /` получает `405`, `HEAD /` получает `200` без тела.
- Оба Telegram-бота успешно проходят `getMe`.
- `npm run check` и `npm audit --omit=dev` проходят; известных npm-уязвимостей нет.
- Playwright: главная страница, модальное окно работы и переключение мобильного режима работают без ошибок консоли.

## Обязательное действие владельца

Telegram-токены и пароль администратора ранее попадали на скриншоты в чате. Они не входят в Git и не отдаются сайтом, однако перед реальным публичным деплоем их следует отозвать/заменить. Новые значения хранить только локально в `.env` и в секретах хостинга.

## Ограничения

Ни один публичный сервис нельзя гарантированно назвать невзламываемым. Для production рекомендуется использовать HTTPS и rate limiting хостинг-платформы, регулярно обновлять Node.js и периодически повторять аудит.
